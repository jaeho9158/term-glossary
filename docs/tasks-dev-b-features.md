# 개발자 B 작업지시서 — 기능 확장

담당 영역: 카테고리 페이지, 검색 고도화, 다크모드, 북마크/학습 기능, 논문 뷰어 고도화, 퀴즈 모드.
피드백 주기가 길 수 있으니, 각 작업은 아래 **완료 기준**을 스스로 체크하고 다음 작업으로 넘어가면 됩니다.

---

## 1. 카테고리 페이지

**현재 문제**: [assets/site.js](../assets/site.js)에 12개 카테고리(`CATEGORY_ORDER`/`CATEGORY_LABELS`: stat, method, tool, ethics, physchem, bioearth, neuro, medhealth, psych, socialecon, eng, cs) 그룹핑 로직이 있지만, index.html 안에서만 보여지고 개별 URL이 없음.

**할 일**
- `category/<code>.html` 형태의 정적 페이지 12개 생성 (또는 `category.html?cat=stat` 같은 쿼리 방식 SPA, 팀 판단에 맡김 — 단 링크 공유 시 카테고리가 바로 보여야 함)
- 데이터 소스는 기존 `terms.json` 그대로 사용, site.js의 그룹핑 로직 재사용
- 각 카테고리 페이지에서 sub_category별로 묶어서 표시 (site.js 로직과 동일)
- index.html의 카테고리 섹션 제목에 "더보기" 링크 추가해서 해당 카테고리 페이지로 연결

**완료 기준**
- 12개 카테고리 페이지 전부 접근 가능, 해당 카테고리 용어 전부 표시됨
- 브라우저에서 직접 URL 입력해서 들어가도 정상 동작 (SPA면 새로고침해도 깨지지 않아야 함)

---

## 2. 검색 고도화

**현재 문제**: 검색이 단순 substring 매칭만 함 (오타 허용 안 됨, 랭킹 없음).

**할 일**
- fuzzy 매칭 도입 (Fuse.js 같은 가벼운 라이브러리, 또는 직접 Levenshtein distance 구현)
- title_ko / title_en / definition 순으로 가중치 다르게 줘서 랭킹 (제목 일치가 정의 본문 일치보다 위로)
- 검색 결과에 카테고리 필터 추가
- 최근 검색어 localStorage에 저장해서 검색창 포커스 시 노출

**완료 기준**
- 오타 1~2글자 있어도 의도한 용어가 상위에 노출됨 (예: "아노바" → "ANOVA" 매칭 확인)
- 제목 일치 결과가 본문 일치 결과보다 항상 위에 뜸

## 3. 다크모드

**할 일**
- style.css를 CSS 변수 기반으로 정리 (색상 하드코딩된 곳 변수로 치환)
- `prefers-color-scheme` 기본 대응 + 우측 상단 토글 버튼으로 수동 전환
- 선택 상태 localStorage에 저장, 새로고침해도 유지

**완료 기준**
- 토글 클릭 시 전체 페이지(헤더/본문/용어 카드/뷰어 포함) 다크 스타일 적용, 대비 확인 (텍스트 가독성)
- 새로고침 후에도 마지막 선택 유지됨

## 4. 북마크 + 개인 학습 기능

**현재 있는 것**: `tg_reading_history` 테이블([assets/term-history.js](../assets/term-history.js))로 열람 이력만 기록됨, 명시적 저장 기능 없음.

**할 일**
- Supabase에 `tg_bookmarks` 테이블 추가 (user_id, term_slug, created_at)
- 용어 페이지에 즐겨찾기 버튼 추가 (로그인 상태에서만 노출, [assets/auth.js](../assets/auth.js) 세션 체크 재사용)
- [history.html](../history.html) 옆에 즐겨찾기 목록 페이지 추가
- **여유 있으면**: 즐겨찾기한 용어를 일정 기간 지나면 "복습 추천"으로 다시 노출하는 기능 (간단한 spaced-repetition 느낌 — 마지막 열람일 + n일 지나면 표시)

**완료 기준**
- 로그인 상태에서 용어 즐겨찾기 추가/해제 정상 동작, DB에 반영 확인
- 즐겨찾기 목록 페이지에서 저장한 용어 전부 확인 가능

## 5. 논문 뷰어 고도화

**현재 있는 것**: [assets/viewer.js](../assets/viewer.js) — PDF 업로드 → 텍스트 추출 → 용어 매칭 하이라이트 → 사이드 term-card, 더블클릭으로 PDF 모달.

**할 일**
- 하이라이트된 용어 클릭 시 사이드 패널에 정의가 바로 뜨도록 (현재 term-card 목록에서 스크롤해서 찾아야 하는지 확인 후, 클릭→해당 카드로 스크롤 또는 팝오버 표시)
- 로그인 사용자 대상으로 PDF 분석 세션 저장/불러오기 (Supabase에 세션 메타데이터 저장, PDF 파일 자체는 저장 안 해도 됨 — 매칭된 용어 목록 + 파일명 정도)
- 여러 PDF 탭으로 동시에 열어서 비교하는 뷰 (선택 사항, 난이도 높음 — 시간 남으면)

**완료 기준**
- 하이라이트 클릭 → 정의 표시까지 3초 이내 체감 반응
- 로그인 후 PDF 분석했다가 페이지 나갔다 다시 들어와도 이전 세션 목록이 history.html류 페이지에서 확인됨

## 6. 퀴즈/플래시카드 모드 (신규 페이지)

**할 일**
- `quiz.html` 신규 페이지: terms.json 기반으로 카테고리 선택 → 랜덤 문제 출제 (정의 보고 용어 맞추기 객관식, 또는 반대로 용어 보고 정의 맞추기)
- 로그인 사용자는 정답률/진행 기록 Supabase에 저장 (`tg_quiz_results` 테이블 등)
- 비로그인도 사용은 가능하되 기록만 저장 안 됨

**완료 기준**
- 카테고리 선택 → 최소 10문제 랜덤 출제 → 채점 결과 표시까지 전체 플로우 동작
- 로그인 상태에서 결과가 DB에 남고, 다시 봤을 때 과거 점수 확인 가능
