# 용어 확장 및 카테고리·검색 UX 설계

## 배경

논문용어사전 사이트([index.html](../../../index.html))는 현재 범용 학술용어 12개를 홈페이지에 평면 목록으로 나열하고 있다. 사용자가 다음 두 가지를 요청했다:

1. 용어를 100개 이상으로 확장한다.
2. 용어가 늘어난 만큼 홈페이지에서 카테고리별로 찾고, 검색창으로도 찾을 수 있게 한다.

이 작업은 이후 별도로 브레인스토밍할 "PDF 업로드 → 원문에서 용어 자동 매칭·하이라이트" 기능의 선행 작업이다. PDF 도구는 이번 계획 범위에 포함하지 않는다.

## 목표

- 용어를 100개 이상으로 확장한다.
- 용어를 다중 태그(카테고리)로 분류할 수 있는 데이터 구조를 도입한다.
- 홈페이지를 카테고리별 섹션 + 클라이언트 사이드 검색으로 재구성한다.
- 기존 12개 용어 페이지와 형식적으로 동일한 신규 용어 페이지를 대량 생성한다.

## 데이터 구조

`terms.json`을 사이트 루트에 새로 만든다. 배열의 각 원소는 다음 형태를 가진다.

```json
{
  "slug": "p-value",
  "title_ko": "유의확률",
  "title_en": "p-value",
  "categories": ["stat", "method"]
}
```

- `slug`: `terms/<slug>.html` 파일명과 일치해야 한다.
- `title_ko`, `title_en`: 홈페이지 목록 표시 및 검색 매칭에 사용한다. `title_en`이 없는 용어(국문 고유 개념 등)는 빈 문자열로 둔다.
- `categories`: 아래 카테고리 코드 중 하나 이상을 포함하는 배열이다. 한 용어가 방법론 카테고리와 분야 카테고리에 동시에 속할 수 있다(예: fMRI는 `["neuro", "tool"]`).

### 카테고리 코드 (12개)

방법론 계열(기존 4개 유지):
- `stat` 통계
- `method` 연구방법론
- `tool` 측정·도구
- `ethics` 윤리·출판

분야 계열(신규 8개):
- `physchem` 물리학·화학
- `bioearth` 생물학·지구과학
- `neuro` 뇌과학·신경과학
- `medhealth` 의학·보건
- `psych` 심리학
- `socialecon` 사회과학·경제학
- `eng` 공학
- `cs` 컴퓨터과학·AI

카테고리 코드와 한글 표시명 매핑은 `terms.json`이 아니라 `index.html`에 내장된 JS 상수로 관리한다(카테고리 자체는 자주 바뀌지 않으므로 별도 파일로 분리하지 않는다).

### 기존 12개 용어 이전

기존 12개 용어 페이지는 그대로 두고, `terms.json`에 각각의 항목을 추가한다. 카테고리는 아래와 같이 배정한다.

| slug | categories |
|---|---|
| p-value | stat, method |
| correlation | stat |
| regression | stat |
| meta-analysis | method, stat |
| sample-size | stat, method |
| confidence-interval | stat |
| significance-level | stat |
| control-group | method |
| cohort-study | method, medhealth |
| rct | method, medhealth |
| qualitative-research | method, socialecon |
| effect-size | stat |

## 홈페이지 재구성

[index.html](../../../index.html)을 다음과 같이 바꾼다.

1. `<ul class="term-list">`로 하드코딩된 12개 링크를 제거한다.
2. 상단에 검색 입력창(`<input type="search" id="term-search">`)을 추가한다.
3. 카테고리 코드 12개 순서대로(방법론 4개 → 분야 8개) `<section>`을 두고, 각 섹션 안에 해당 카테고리를 가진 용어를 `terms.json`에서 필터링해 렌더링한다. 한 용어가 여러 카테고리에 속하면 해당하는 모든 섹션에 중복 표시된다.
4. 렌더링과 검색 로직은 `assets/site.js`라는 새 파일로 분리한다(인라인 스크립트로 넣지 않는다).
5. 검색 동작: 입력값이 바뀔 때마다 `title_ko` 또는 `title_en`에 입력값(대소문자 무시)이 포함된 용어만 남기고 나머지는 각 섹션에서 숨긴다. 검색어가 비어 있으면 전체 카테고리 뷰로 복귀한다. 검색 결과가 0개인 섹션은 섹션 제목까지 숨긴다.
6. 정적 페이지 자체는 JS 없이도 `terms.json` 데이터가 사람이 읽을 수 있는 형태이므로, JS가 실행되지 않는 환경(검색엔진 크롤러 등)에서도 `<noscript>` 블록에 전체 용어를 한 번에 나열해 SEO 크롤링 가능성을 유지한다.

## 신규 용어 87개 이상 확보

기존 12개에 더해 88개 이상의 신규 용어를 추가해 합계 100개 이상을 만든다(여유를 두어 목표를 88개로 잡는다). 각 신규 용어는 방법론 4개 카테고리 우선으로 최소 24개를 채우고, 나머지를 8개 분야 카테고리에 분배한다. 정확한 용어 목록은 계획(plan) 작성 단계에서 카테고리별로 확정한다.

각 용어 페이지는 기존 12개와 동일한 HTML 템플릿([terms/p-value.html](../../../terms/p-value.html) 구조 참고)을 따른다: breadcrumb, h1, definition-box(한 줄 정의), 쉬운 풀이 섹션, 논문 예문(example 박스), 주의할 점, 관련 용어 링크 2~3개. `<title>`과 `<meta description>`도 용어별로 채운다.

## 자기 검토

- **placeholder 스캔**: 없음. 신규 용어 목록은 계획 단계에서 구체화하기로 명시했으며 이는 미정이 아니라 "계획 문서의 책임 범위"로 의도적으로 넘긴 것이다.
- **내부 일관성**: `terms.json`의 카테고리 코드와 홈페이지 섹션 코드가 1:1로 매핑되어야 한다는 점을 명시했다.
- **범위 점검**: PDF 도구는 명시적으로 범위 밖으로 뺐다. 단일 계획으로 실행 가능한 크기다.
- **모호성 점검**: 검색 매칭 기준(부분 문자열, 대소문자 무시)과 다중 카테고리 표시 방식(중복 노출)을 명시했다.
