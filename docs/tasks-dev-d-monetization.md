## 1. Google Search Console 등록 확인 및 정비

**현재 상태**: [googlebece5485f861324c.html](../googlebece5485f861324c.html) 파일이 이미 존재 — 과거에 소유권 인증을 했거나 시도한 흔적. 실제로 Search Console에 등록되어 데이터가 쌓이고 있는지부터 확인 필요.

**할 일**
- Search Console(search.google.com/search-console)에 로그인해서 이 속성이 실제 등록되어 있는지 확인
- 등록 안 되어 있으면 기존 인증 파일로 재인증 시도, 안 되면 새로 인증 파일 발급받아 루트에 추가
- sitemap.xml을 Search Console에 제출 (URL 검사 도구로 몇 개 용어 페이지 색인 상태 확인)
- **주의**: 지금 sitemap.xml에 용어 페이지가 다 포함되어 있는지는 개발자 A가 작업 중(`generate-sitemap.js`) — 그 작업 끝난 뒤에 제출하는 게 효율적. 순서 조율 필요
- 색인 커버리지 리포트에서 제외/오류 있는 페이지 확인, 있으면 원인 파악해서 개발자 A/B에게 전달

**완료 기준**
- Search Console에서 사이트 소유권 인증 완료 상태 확인 스크린샷
- sitemap.xml 제출 완료, "제출됨" 상태 확인
- 색인된 페이지 수가 실제 용어 페이지 수(200개+)에 근접

## 2. 네이버 서치어드바이저 등록

**배경**: 한국어 용어 사전 사이트라 네이버 검색 유입이 구글보다 클 가능성이 높음. 네이버는 별도 등록/최적화가 필요.

**할 일**
- 네이버 서치어드바이저(searchadvisor.naver.com)에 사이트 등록, HTML 태그 또는 파일 방식으로 소유 확인
- 네이버용 sitemap 제출 (기존 sitemap.xml 그대로 제출 가능, 형식 호환됨)
- robots.txt에 네이버 봇(Yeti) 관련 이슈 없는지 확인 (현재 `Allow: /`라 문제 없을 것으로 보이나 재확인)
- 네이버 웹마스터 가이드 기준으로 메타 description, title 태그 최적화 상태 점검 (용어 페이지들의 title/description이 각기 다르게 잘 채워져 있는지 확인 — 중복 title은 감점 요인)

**완료 기준**
- 네이버 서치어드바이저 소유 확인 완료
- 사이트: 검색 시 네이버에서 결과 노출 확인 (등록 직후엔 시간 걸릴 수 있음, 1~2주 내 재확인)

## 3. 사전 점검: 애드센스 승인 요건 확인

**중요**: 애드센스는 콘텐츠 품질/양, 오리지널리티, 정책 준수를 심사함. 반려되면 재신청까지 대기 기간이 있으므로 신청 전에 아래를 먼저 점검.

**할 일**
- 콘텐츠 양 확인: 구글 애드센스는 명확한 최소 개수를 공표하진 않지만, 실무적으로 최소 20~30개 이상의 충실한 페이지가 권장됨 — 현재 210개 용어 페이지 보유로 양적 요건은 충족 가능성 높음, 다만 각 페이지 정의가 너무 짧으면(1문장 수준) 콘텐츠 빈약 사유로 반려될 수 있음 → 개발자 C의 용어 콘텐츠 보강 작업과 연계해서 확인
- 필수 페이지 존재 확인: [about.html](../about.html), [privacy.html](../privacy.html), [contact.html](../contact.html) — 이미 있음, 개인정보처리방침 내용이 실제 애드센스/쿠키 사용을 반영하고 있는지 privacy.html 내용 검토 (애드센스 붙이면 쿠키 고지 필수)
- 사이트가 크롤링/정상 로딩되는지 확인 (JS 에러로 콘텐츠 안 보이면 심사 실패 요인)
- GitHub Pages 서브도메인(`jaeho9158.github.io`)으로 애드센스 신청이 가능한지 최신 정책 확인 — 커스텀 도메인이 없으면 승인이 더 까다로울 수 있으므로, 예산이 허용되면 커스텀 도메인 구매를 먼저 검토하고 사용자(대표)에게 도메인 구매 여부 확인받기

**완료 기준**
- 위 점검 항목 체크리스트로 정리해서 보고, 미흡한 부분은 목록화

## 4. 애드센스 신청

**주의**: 이 단계는 실제 신청 제출이 포함되므로, 3번 점검이 끝난 뒤 진행 여부를 먼저 공유하고 진행할 것.

**할 일**
- Google AdSense(adsense.google.com) 계정 생성, 사이트 추가
- 애드센스 코드 스니펫을 전체 페이지 `<head>`에 삽입 (개발자 A가 진행 중인 애널리틱스 스니펫 삽입 작업과 같은 파이프라인 사용 — 같이 처리하면 효율적, 조율 필요)
- 심사 제출 후 결과 대기 (통상 며칠~몇 주)
- 반려 시 사유 확인해서 정리, 재신청 전 보완 작업 항목 목록화

**완료 기준**
- 애드센스 신청 제출 완료 확인 화면
- 승인/반려 결과 및 (반려 시) 사유 정리 보고
